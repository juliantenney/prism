## Scope statement

This document defines **instructional episode archetypes** by cognitive demand for **self-directed** learning. An episode is a **coherent sequence of instructional moves** — not a material-type checklist, not an ACM component list, and not a page layout.

**Guiding question answered:**

> What sequence of instructional functions best supports learning at different cognitive demands when **no tutor** can recover thin materials?

---

## Section 1 — Episode design principles

### 1.1 What counts as an episode

An **instructional episode** is the smallest unit of coached learning that:

1. Has a **clear intellectual purpose** tied to one primary cognitive demand.  
2. Moves the learner through a **recognisable sequence** of instructional functions — not a single task shell.  
3. Produces **observable evidence** that the demand was met (not only that a table was filled).  
4. Ends with **closure** — verification, reflection, or transition — before the next episode begins.

A workbook **session** is typically **3–6 episodes** (38C-1). Each episode may span one activity block or a tightly coupled pair (e.g. model + parallel practice).

### 1.2 Self-directed design rules (from 28–31, 38C)

| Principle | Implication for episodes |
|-----------|-------------------------|
| **No tutor recovery** | Every function in the sequence must be **realised in artefacts** — orientation, modelling, checks, repair paths. |
| **Visible pedagogy ≠ episode richness** | Framing and preambles are necessary but **insufficient**; episodes must include teaching and verification moves. |
| **Structure ≠ pedagogy** | Correct workflow topology and material types do not imply a coached episode. |
| **Support must not thin as demand rises** | Evaluate episodes need **more** instructional moves than Understand — not fewer (38G-2; H-04). |
| **Scaffold fading is session-level** | Within an episode, support peaks at modelling/guided practice and reduces at independent performance; **across** a session, later episodes assume more independence. |
| **Non-spoiler closure** | Reflection and consolidation require **learner-generated** synthesis — not model essays (38H-2). |
| **Explicit beats ordinary briefs** | Episode patterns are the **default design target**, not optional rhetoric when the brief is vague (28 H11/H13). |

### 1.3 Instructional move vocabulary

Moves are drawn from [38I-1 §8](38I-1-prior-pedagogical-journey-review.md). Names used in this model:

| Move | Short definition |
|------|------------------|
| **Orientation** | Why this episode now; link to session arc |
| **Framing** | Bounded problem space, stakes, context |
| **Activation** | Elicit prior knowledge relevant to the episode |
| **Explanation** | Teach concepts/relationships needed for success |
| **Example** | Positive instance illustrating the target idea |
| **Non-example** | Contrast case that clarifies boundaries |
| **Misconception challenge** | Surface tempting error; prompt reconciliation with evidence |
| **Criteria exposition** | Make rules, dimensions, or comparison lenses explicit |
| **Worked thinking** | Expert reasoning visible step-by-step (not only final answer) |
| **Guided inquiry** | Bounded exploration with prompts sustaining productive uncertainty |
| **Guided practice** | Learner attempt with heavy scaffolding (hints, partial structure) |
| **Independent practice** | Learner attempt with reduced scaffolding |
| **Verification** | Check understanding before continuing (self-check, keyed criteria) |
| **Revision** | Compare attempt to evidence/model; update reasoning |
| **Reflection** | Articulate meaning in own words (self-explanation) |
| **Synthesis** | Integrate multiple ideas into one coherent product |
| **Evaluative judgement** | Compare options and justify using criteria/exemplars |
| **Transfer** | Apply to learner’s own context with explicit criteria |
| **Transition** | Bridge to next episode; state what was established |

**Classification key (used in Sections 2–6):**

| Tag | Meaning |
|-----|---------|
| **R** | **Required** — high-quality episode normally fails without this move |
| **C** | **Common** — frequently valuable; context-dependent |
| **O** | **Optional** — strengthens episode when topic/risk warrants |

### 1.4 Episode progression dimensions

How episodes differ **across cognitive demands** (summary; detail per archetype below):

| Dimension | Understand | Apply | Analyse | Evaluate |
|-----------|------------|-------|---------|----------|
| **Scaffold intensity** | High through explanation; moderate at classification | High at worked thinking; fades to independent | High at criteria + worked analysis; fades at justification | High at criteria/exemplars; moderate at judgement (criteria stay visible) |
| **Learner independence** | Low → moderate | Moderate → high | Moderate → high | High (learner owns criteria application) |
| **Role of modelling** | Concept instances | **Process** modelling central | **Reasoning pattern** modelling central | **Quality judgement** modelling (exemplar contrast) |
| **Role of feedback** | Corrective (misconception repair) | Procedural (did steps match method?) | Evidential (does analysis cite right relationships?) | Criteria-based (strength of justification) |
| **Role of verification** | Recognition / classification checks | Performance match to worked example | Structured justification checkpoints | Rubric-aligned self-audit |
| **Role of transfer** | Usually **O** (bounded analogy) | **C** (new context, same method) | **C** (new case, same analysis frame) | **R** (judgement in authentic or personal context) |
| **Role of reflection** | **C** — consolidate meaning | **C** — what was tricky in the method | **R** — explain why analysis holds | **R** — defend judgement under trade-offs |

## Section 2 — Understand episode archetype

### 2.1 Episode purpose profile

| Field | Content |
|-------|---------|
| **Primary learning purpose** | Build accurate mental models — concepts, definitions, distinctions, and relationships sufficient for later Apply/Analyse work |
| **Typical learner challenge** | Confusing similar ideas; accepting definitions without discrimination; carrying misconceptions forward |
| **Instructional response** | Teach → exemplify → contrast → confront misconceptions → check discrimination → briefly consolidate |
| **Expected learner output** | Correct classification or explanation of concepts; completed self-check showing discrimination; short reflection or summary in own words |

### 2.2 Canonical function sequence

Evidence basis: 28 activate→confront→reconcile; 38C explanatory teaching + retrieval; charter Understand draft; 38I-1 §8.1–8.2.

```text
Orientation
  → Framing
  → Activation
  → Explanation
  → Example
  → [Non-example]
  → [Misconception challenge]
  → [Guided classification]
  → Independent classification
  → Verification
  → [Reflection]
  → Transition
```

**Narrative:** Orient and frame the ideas → activate prior knowledge → teach the concept(s) → show what counts and what does not → optionally repair a tempting error → guide then release classification or explanation → verify discrimination → optionally reflect → bridge forward.

### 2.3 Function classification — Understand

| Function | Tag | Rationale |
|----------|-----|-----------|
| Orientation | **R** | Solo learners skip teaching without purpose cue |
| Framing | **C** | Strong when concepts are abstract or easily conflated |
| Activation | **C** | High value when episode builds on prior session content |
| Explanation | **R** | Core teaching move — 38C R1 explanatory teaching |
| Example | **R** | Makes definition concrete |
| Non-example | **C** | Critical when confusable pairs exist (e.g. CPI vs GDP deflator) |
| Misconception challenge | **C** | **R** when KM lists misconceptions for mapped concepts |
| Guided classification | **C** | Helps first discrimination attempt |
| Independent classification | **R** | Observable evidence of understanding |
| Verification | **R** | 38C retrieval minimum — at least one check episode |
| Reflection | **C** | Self-explanation solidifies mental model |
| Transition | **C** | Links to Apply/Analyse episodes that follow |

### 2.4 Progression characteristics — Understand

- **Scaffold intensity:** Highest during Explanation and Example; Misconception challenge and Guided classification carry support into first performance.  
- **Modelling:** Models **what the concept is/is not** — not yet full procedural method.  
- **Revision:** **O** — mainly “reclassify after contrast”; full revision cycles belong in Apply+.  
- **Transfer:** **O** — light analogy only; deep transfer deferred.

## Section 3 — Apply episode archetype

### 3.1 Episode purpose profile

| Field | Content |
|-------|---------|
| **Primary learning purpose** | Execute a method, procedure, or rule in a structured context — reproduce expert process with decreasing help |
| **Typical learner challenge** | Knowing definitions but not **how** to proceed; skipping steps; applying wrong procedure to context |
| **Instructional response** | Contextualise → explain process → model worked thinking → guided then independent practice → verify against method → reflect on process |
| **Expected learner output** | Completed worksheet/table/calculation/scenario response showing **steps**; self-check against worked example or checklist |

### 3.2 Canonical function sequence

Evidence basis: 38C worked examples + guided practice; 28 worked→parallel task; charter Apply draft; 38E-8 worked_example/sample_output retention.

```text
Orientation
  → Framing
  → [Activation]
  → Criteria exposition (process/rules)
  → Worked thinking
  → Guided practice
  → Independent practice
  → Verification
  → [Revision]
  → [Reflection]
  → [Transfer]
  → Transition
```

**Narrative:** State what skill is being applied and in what context → make rules/steps explicit → model one full expert run → learner tries with hints → learner tries with reduced help → verify procedure and result → optionally revise against model → reflect on process → optionally apply to adjacent context → bridge.

### 3.3 Function classification — Apply

| Function | Tag | Rationale |
|----------|-----|-----------|
| Orientation | **R** | Links application to session arc |
| Framing | **R** | Apply requires situational context (scenario, data, case) |
| Activation | **O** | Brief recall of prerequisite concepts |
| Criteria exposition | **R** | Method/rules must be explicit before practice |
| Worked thinking | **R** | 38C R5 — modelling path mandatory for workbook quality |
| Guided practice | **R** | Scaffolded first attempt |
| Independent practice | **R** | Evidence of application without full scaffolding |
| Verification | **R** | Checklist, keyed steps, or comparison to sample structure |
| Revision | **C** | Compare attempt to worked example; update steps |
| Reflection | **C** | “What step was hardest?” — metacognitive hook |
| Transfer | **C** | Same method, new numbers/context (38C transfer partial path) |
| Transition | **C** | Preview Analyse/Evaluate use of the skill |

### 3.4 Progression characteristics — Apply

- **Scaffold intensity:** Peaks at Worked thinking and Guided practice; **fade** is the defining episode arc.  
- **Modelling:** **Process-centred** — visible steps and decisions.  
- **Feedback:** Procedural — “did you follow the method?” more than “is your opinion valid?”  
- **Revision:** **Common** when initial attempt diverges from model — 28 revision cycle at intro level.  
- **Transfer:** **Common** — new surface context, same procedure.

## Section 4 — Analyse episode archetype

### 4.1 Episode purpose profile

| Field | Content |
|-------|---------|
| **Primary learning purpose** | Decompose, compare, classify, or relate parts using explicit criteria — see structure, not only recall or execute |
| **Typical learner challenge** | Jumping to conclusions; analysing without criteria; spotting features but not relationships |
| **Instructional response** | Frame analytical question → expose criteria/lenses → model analytic reasoning on one case → guide structured analysis → independent analysis with justification → verify logic → reflect on reasoning |
| **Expected learner output** | Completed analysis table/matrix; written justification citing relationships; structured comparison with explicit dimensions |

### 4.2 Canonical function sequence

Evidence basis: 28 D1/D4 productive uncertainty; 38C evaluative judgement partial; charter Analyse draft; 38G-2 Analyse bundle (relations/processes).

```text
Orientation
  → Framing (analytical question)
  → [Activation]
  → Criteria exposition (dimensions/lenses)
  → Explanation (of relationships relevant to analysis)
  → Worked thinking (analytic walkthrough)
  → Guided inquiry
  → Guided practice (structured analysis)
  → Independent practice (analysis + justification)
  → Verification
  → Reflection
  → [Transfer]
  → Transition
```

**Narrative:** Pose what is being analysed and why → make comparison/classification criteria explicit → teach needed relational ideas → model one analytic pass → guided prompts through uncertain middle → structured partial then full analysis → check logic and evidence use → learner explains reasoning → optional new case → bridge.

### 4.3 Function classification — Analyse

| Function | Tag | Rationale |
|----------|-----|-----------|
| Orientation | **R** | Analysis tasks look like “answer questions” without purpose |
| Framing | **R** | Bounded analytical question with stakes |
| Activation | **C** | Recall concepts/relationships to be used |
| Criteria exposition | **R** | Analysis without lenses collapses to opinion |
| Explanation | **C** | **R** when relationships/processes are non-obvious |
| Worked thinking | **R** | Model **analytic** steps, not only final classification |
| Guided inquiry | **C** | Sustains productive uncertainty (28 D3) |
| Guided practice | **R** | Partially filled matrix/table with hint column |
| Independent practice | **R** | Full analysis + justification |
| Verification | **R** | Criteria checklist or “evidence cited?” audit |
| Reflection | **R** | Self-explanation of why analysis holds |
| Transfer | **C** | New case, same criteria frame |
| Transition | **C** | Link to Evaluate if judgement follows |

### 4.4 Progression characteristics — Analyse

- **Scaffold intensity:** High at criteria + worked analytic model; Guided inquiry optional bridge; independence at justified analysis.  
- **Modelling:** **Reasoning-pattern-centred** — how to move from evidence to structured conclusion.  
- **Feedback:** Evidential and structural — relationships invoked correctly?  
- **Revision:** **C** — update analysis after contrast case or misconception challenge.  
- **Transfer:** **Common** — same analytic frame, new dataset/scenario.

## Section 5 — Evaluate episode archetype

### 5.1 Episode purpose profile

| Field | Content |
|-------|---------|
| **Primary learning purpose** | Judge quality, weigh trade-offs, and defend decisions using explicit criteria — not only analyse parts |
| **Typical learner challenge** | Listing pros/cons without criteria; premature closure; unsupported recommendations; capstone tasks that repeat analysis without judgement |
| **Instructional response** | Present authentic decision context → surface competing perspectives → teach decision criteria and trade-offs → model quality judgement → guided evaluation → independent judgement task → verify against rubric → reflect on trade-offs → transfer to learner context |
| **Expected learner output** | Ranked/selected option with **justification**; decision memo; rubric-scored self-audit; personal recommendation with criteria cited |

**Programme gap:** H-04 — Evaluate LO on inflation capstone without dedicated evaluative episode. This archetype is the **target state** 38I-4 should exemplify.

### 5.2 Canonical function sequence

Evidence basis: 38C §3.11 evaluative judgement; 28 competing perspectives; charter Evaluate draft; 30 reasoning/transfer on Marx probe; 38I-1 §6.2 Evaluate gap.

```text
Orientation
  → Framing (decision/judgement context)
  → Activation
  → Criteria exposition (decision rules + trade-off dimensions)
  → Explanation (of perspectives or constraints)
  → [Worked thinking — exemplar judgement]
  → Guided inquiry (explore trade-offs)
  → Guided practice (partial evaluation)
  → Independent practice (judgement task)
  → Evaluative judgement
  → Verification (rubric/quality audit)
  → Reflection
  → Transfer
  → Transition
```

**Narrative:** Clarify what is being decided and why it matters → make criteria and trade-offs explicit → teach background needed for informed judgement → optionally model strong vs weak judgement → explore tensions → scaffold partial evaluation → learner produces defended judgement → self-audit against rubric → reflect on trade-offs → apply to personal/professional context → close session arc.

### 5.3 Function classification — Evaluate

| Function | Tag | Rationale |
|----------|-----|-----------|
| Orientation | **R** | Evaluate tasks are often mistaken for summary activities |
| Framing | **R** | Authentic decision space with constraints |
| Activation | **C** | Integrate prior episode concepts |
| Criteria exposition | **R** | **Core** — without criteria, evaluation becomes opinion |
| Explanation | **C** | Perspectives/constraints when topic is contested |
| Worked thinking (exemplar judgement) | **C** | Strong vs weak exemplar — not spoiler answer for learner task |
| Guided inquiry | **C** | Trade-off exploration (28 productive uncertainty) |
| Guided practice | **C** | Partial ranking with justification prompts |
| Independent practice | **R** | Learner-owned judgement artefact |
| Evaluative judgement | **R** | Distinct move — compare options using criteria |
| Verification | **R** | Rubric, quality checklist, exemplar contrast |
| Reflection | **R** | Metacognitive: what would change your judgement? |
| Transfer | **R** | 38C R6 path — personal/professional application |
| Transition | **O** | Session closure often coincides with episode end |

### 5.4 Progression characteristics — Evaluate

- **Scaffold intensity:** High early on **criteria and trade-offs**; remains available during judgement — criteria do not fade away as in Apply.  
- **Learner independence:** Highest — learner must **own** the recommendation.  
- **Modelling:** **Quality-centred** — exemplar contrast, not full spoiler solution.  
- **Feedback:** Criteria-aligned — strength of justification, not single right answer (unless brief is closed).  
- **Revision:** **C** — update judgement after new evidence or self-audit.  
- **Transfer:** **Required** for workbook-class Evaluate episodes (household budget, policy stance, professional practice).  
- **Metacognitive support:** **R** via Reflection — addresses deferred 30 `metacognition_contract` at episode level without new schema.

| Activation | C | O | C | C |
| Explanation | R | C | C | C |
| Example | R | — | C | C |

| Criteria exposition | O | R | R | R |

| Revision | O | C | C | C |
| Reflection | C | C | R | R |

### 6.2 Progression dimension summary

| Dimension | Understand | Apply | Analyse | Evaluate |
|-----------|------------|-------|---------|----------|
| Scaffold intensity (peak → release) | High → moderate | High → low | High → moderate | High (criteria) → moderate |
| Learner independence (end state) | Moderate | High | High | Highest |
| Modelling focus | Concept instances | Procedure | Analytic reasoning | Judgement quality |
| Verification focus | Discrimination | Procedure match | Logic/evidence | Rubric/criteria |
| Transfer | Optional | Common | Common | **Required** |
| Reflection depth | Light | Process | Reasoning | Trade-offs |

### 6.3 Session composition guidance (non-normative)

Typical **60-minute self-study workbook** (38C-1) — episode **mix**, not prescription:

| Session arc position | Likely primary demand | Notes |
|---------------------|----------------------|-------|
| Early episodes | Understand → Apply | Establish concepts then procedural fluency |
| Middle episodes | Apply → Analyse | Scenarios, tables, comparison tasks |
| Late episodes | Analyse → Evaluate | Integration, judgement, personal transfer |
| Closure | Evaluate or Analyse+Reflection | Non-spoiler synthesis; avoid model essay |

**Anti-pattern:** Four Apply shells with no Understand teaching or Evaluate judgement — matches 38F “assembled” workbook failure.

### 6.4 Additional cognitive levels

| Level | Verdict |
|-------|---------|
| **Remember** | **Not separate archetype** — folded into **Activation** and **Verification** within Understand/Apply episodes. |
| **Create** | **Not separate archetype** — treated as **Synthesis** (optional C) within Analyse/Evaluate when the brief requires original design or novel combination. Add dedicated archetype only if 38I-4 mock-ups show repeated gap. |

## Section 7 — Implications for 38I-3 (KM/LO → episode mapping)

38I-3 should map **existing** KM and LO affordances to episode functions — **no schema expansion**.

### 7.1 KM affordances → episode functions (mapping intent)

| KM affordance | Primary episode functions enabled | Typical cognitive demand |
|---------------|-----------------------------------|--------------------------|
| **Concepts + definitions** | Explanation, Example, Activation | Understand |
| **Relationships** | Explanation, Criteria exposition, Guided/Independent practice | Analyse |
| **Processes** | Criteria exposition, Worked thinking, Guided practice | Apply, Analyse |
| **Groupings / hierarchies** | Framing, Explanation, Guided classification | Understand, Analyse |
| **Misconceptions** | Misconception challenge, Non-example, Verification | Understand, Analyse, Evaluate |

**38I-3 task:** For each archetype, specify **which KM slots must be read** before authoring the episode sequence (e.g. Evaluate episode requires misconceptions + relationships for trade-off reasoning).

### 7.2 LO properties → episode selection and density

| LO property | Mapping intent |
|-------------|----------------|
| **Cognitive level** (Understand / Apply / Analyse / Evaluate) | **Primary archetype selector** — one episode’s main demand |
| **Concept mappings** | Scope for Explanation, Example, Misconception challenge |
| **Progression** (session position) | Scaffold fading across episodes; Transition content |
| **Outcome intent** | Expected learner output shape (classification vs procedure vs justification) |

**38I-3 task:** Define rules such as: “Evaluate LO → Evaluate archetype **required**; Apply-only episode is **non-compliant**.”

### 7.3 Information 38I-3 must produce

| Output | Purpose |
|--------|---------|
| **Archetype selection rule** | LO cognitive level → primary episode archetype |
| **KM trigger table** | Which KM slots force **R** moves (e.g. misconceptions → Misconception challenge) |
| **Minimum move set per archetype** | Machine-checkable checklist for 38I-4 mock-up review |
| **Multi-LO episodes** | How to combine when one activity maps to several LOs |
| **Gap register** | Where KM/LO **do not** supply enough information for a move (author must infer from brief — document risk) |

### 7.4 Explicit non-goals for 38I-3

- No new KM/LO fields  
- No ACM redesign — ACM may be **referenced** as historical alignment only  
- No DLA/GAM field mapping (deferred to 38I-5 implementation implications)  
- No renderer or presentation criteria  

### 7.5 Open questions for 38I-3 / 38I-4

1. When LO level is **Evaluate** but KM lacks misconceptions, is Misconception challenge **O** or replaced by trade-off inquiry?  
2. How should **multi-concept** Understand episodes split Example/Non-example beats?  
3. Minimum **Verification** depth per archetype — checklist vs keyed answers vs exemplar contrast?  
4. **Synthesis** vs **Evaluate** boundary when capstone is integrative plan (inflation household budget).
