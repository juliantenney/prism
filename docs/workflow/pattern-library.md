# Domain Pattern Library

Current focus: **Pedagogy**

Add new domains over time as they emerge.

## Pattern: Lesson Plan Builder

Use when:
- You need a lesson from source content and a learner profile.

Typical inputs:
- Source content
- Learner level
- Duration
- Delivery format

Typical steps:
1. Extract key concepts
2. Define measurable outcomes
3. Build lesson flow
4. Create activities
5. Add assessment
6. Add instructor notes

Suggested prompt tags:
- `pedagogy`, `lesson-design`, `activities`, `assessment`

## Pattern: Instructor Guide from Raw Content

Use when:
- You have rough material and need facilitator-ready guidance.

Typical inputs:
- Raw text/content dump
- Session objective
- Time budget
- Audience profile

Typical steps:
1. Normalize and structure source content
2. Map content to outcomes
3. Draft instructional sequence
4. Build facilitation script
5. Build learner handout cues
6. QA for alignment/time/accessibility

Suggested prompt tags:
- `pedagogy`, `instructor-guide`, `facilitation`, `alignment`

## Pattern: Outcome-Assessment Alignment Pack

Use when:
- You need rigorous alignment between outcomes, activities, and assessments.

Typical inputs:
- Draft outcomes
- Existing activities and assessments
- Standards/rubric requirements

Typical steps:
1. Evaluate outcome quality
2. Audit alignment matrix
3. Propose revisions
4. Rebuild assessments
5. Generate final alignment report

Suggested prompt tags:
- `pedagogy`, `alignment`, `assessment`, `rubric`

## Pattern: Microlearning Module (Short Duration)

Use when:
- You need 5-15 minute learning segments.

Typical constraints:
- Tight time box
- Few objectives
- Strong practical focus

Typical steps:
1. Narrow outcomes to essentials
2. Design compact activity
3. Build quick formative check
4. Generate concise facilitator instructions

Suggested prompt tags:
- `pedagogy`, `microlearning`, `activities`

## How to Extend

For each new domain pattern, include:
