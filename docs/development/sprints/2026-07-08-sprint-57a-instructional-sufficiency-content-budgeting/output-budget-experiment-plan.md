# Output-Budget Experiment Plan

## Experiment order

1. Define learner-time and content-budget assumptions.
2. Constrain DLA activity design accordingly.
3. Constrain GAM material generation accordingly.
4. Generate a complete page.
5. Audit instructional validity.
6. Only then evaluate output-size/model-budget constraints.
7. Gradually increase richness to find the real failure boundary.

## Decision criterion

Not: "How do we work around output limits?"  
But: "Can PRISM produce instructionally valid materials within realistic learner-time and content budgets?"

## Outputs

- assumption set versioned in 57A docs
- experimental run log per page
- instructional validity audit notes
- boundary findings (where quality or feasibility degrades)
