# Sprint 68 — Acceptance Criteria

High-level acceptance targets. Refine after INV-001 completes.

---

## Learner experience

1. Consecutive activities feel connected where authoritative data supports transitions; abrupt context changes reduced accordingly.
2. Bridge / preamble / progression content appears in pedagogically appropriate positions when data exists.
3. Heteroscedasticity export reads as a coherent self-study lesson at desktop and mobile widths where the lesson model supplies sufficient information.

## Boundary validation

4. Each investigated coherence gap is classified: renderer defect, render model deficiency, lesson schema deficiency, or pipeline generation deficiency.
5. Deferred improvements are documented with evidence when pedagogical information is absent from the lesson model — deferral is acceptable success.
6. Deferred schema register entries include evidence, learner impact, and why renderer improvements alone are insufficient.

## Technical

7. All renderer changes derive from existing JSON / model fields unless deferred register documents otherwise.
8. vNext regression suite remains green.
9. Legacy explicit path unchanged unless intentionally tested.
10. No Sprint 67 shell, navigation, or CSS regressions.

## Documentation

11. Investigation log and findings log updated with ownership classification.
12. Implementation notes capture placement decisions and boundary conclusions.
13. Closure document records render model gaps and pipeline recommendations.
