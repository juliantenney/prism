## Mission

Determine the **target-state Design Page architecture** and produce an **approved migration plan** before any implementation sprint begins.

---

## Scope

| In scope | Detail |
| -------- | ------ |
| Consolidate investigation findings | Failure modes, audit, remediation evidence |
| Responsibility mapping | Transport vs organise vs author vs present vs workflow |
| Target-state architecture definition | Transport-first model; optional bridge layer |
| Migration sequencing | Dependencies, risks, validation gates |
| Sprint 57 impact assessment | Defer, narrow, or sequence relative to 56A outcomes |
| Open question resolution | Document decisions or explicit deferrals |

---

## Non-goals

Sprint 56A is explicitly **not**:

| Non-goal | Reason |
| -------- | ------ |
| Prompt rewrite sprint | Implementation follows approved plan |
| Prompt rationalisation sprint | Size reduction without responsibility reduction is insufficient |
| Learner experience sprint | UX work depends on trustworthy page payloads |
| Implementation sprint | No code, pack, or prompt changes in 56A |
| Reopening full Sprint 56 programme | Targeted Design Page scope only |
| Duplicating Sprint 57 package | Reference and inherit |

6. [ ] Implementation sprint chartered with scope bounded to approved migration plan
