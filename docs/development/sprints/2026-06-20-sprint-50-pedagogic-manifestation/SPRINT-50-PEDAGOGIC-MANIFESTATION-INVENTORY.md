## 8. Duplication, overlap, and dead fields

### Dual render paths (same field, two components)

| Field | Path A | Path B |
| ----- | ------ | ------ |
| `reasoning_orientation` | Framing: “How to think” cue | Journey Compass signal — not `util-cognition` |
| `self_explanation_prompt` | `util-cognition--explain` | Compass “Reflect”; **not** framing block |
| `transfer_or_application_task` | `util-cognition--transfer` | Compass “Apply”; may also exist as `materials.transfer_prompt` (material-level) |
| `uncertainty_tension_prompt` | `util-cognition--uncertainty` | Compass “Uncertainty” |
| `study_orientation`, `intellectual_frame`, `intellectual_coherence_bridge` | Framing block | Compass signals |

### Alias overlap (single render path, multiple keys)

- `activity_preamble` / `orienting_preamble` / `activity_framing`
- `prior_knowledge_activation` / `prior_knowledge_prompt`
- `reasoning_orientation` / `reasoning_orientation_prompt`
- `support_note` / `support_notes`

### Framing vs cognition split

- **Framing block** (`util-activity-framing`): orientation + reasoning_orientation + argument/conceptual/evidence cues
- **Cognition block** (`util-cognition`): pack fields, self_explanation, transfer_or_application, uncertainty, scaffold_hint_sequence
- **Overlap risk:** `reasoning_orientation` never enters `util-cognition`; `argument_structure_hint` and `conceptual_contrast_prompt` only in framing, not cognition groups

### Material vs row transfer

- `transfer_or_application_task` (DLA row) vs `materials.transfer_prompt` (GAM body) — both learner-visible; Compass may pointer-reference material

### Dead / legacy / facilitator-only

- `orienting_preamble`, `activity_framing` — legacy aliases, still honoured
- `facilitator_note(s)`, `teacher_notes`, `instructor_guidance` — facilitator; sanitized off learner rows
- Deprecated rhetoric builder functions — all route to `buildLdSelfDirectedRhetoricPromptBlock`

### Compose fidelity gap (Marx run2)

- DLA generates full PEL set; Design Page `page.json` may omit PEL keys while `materials` are preserved
- HTML with preamble present implies either merged compose validation or render-with-upstream — not inferable from `page.json` alone
