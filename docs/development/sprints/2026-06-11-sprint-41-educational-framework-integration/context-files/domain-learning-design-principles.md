# domain-learning-design-principles.md

## Purpose

This document defines the core instructional design principles that should guide the creation of learning design workflows within PRISM.

These principles act as **constraints and quality criteria** for workflow generation.  
They should be applied consistently across all steps, prompts, and artefacts.

---

# 1. Constructive Alignment

Learning design must ensure alignment between:

- learning outcomes  
- learning activities  
- assessment  

### Requirements

- Every outcome must be supported by at least one activity.
- Every assessed item must map to a defined outcome or concept.
- Activities and assessments should reflect the same level of cognitive demand.

---

# 2. Measurable Learning Outcomes

Learning outcomes must describe **observable and assessable performance**.

### Requirements

- Avoid vague verbs such as:
  - understand  
  - know  
  - appreciate  

- Prefer verbs that imply observable performance:
  - explain  
  - analyse  
  - evaluate  
  - apply  
  - construct  

- Outcomes must:
  - be specific  
  - be assessable  
  - relate directly to content  

---

# 3. Active Learning

Learners should be required to **do something meaningful with knowledge**.

### Requirements

- Activities must involve:
  - application  
  - interpretation  
  - comparison  
  - problem-solving  

- Avoid passive-only designs (e.g. read/watch only).
- Tasks should require cognitive engagement, not just recall.

---

# 4. Valid Assessment

Assessment must provide **credible evidence of learning**.

### Requirements

- Assessment tasks must:
  - directly test intended outcomes or concepts  
  - reflect the level of performance expected  

- Avoid:
  - trivial recall-only questions (unless explicitly required)
  - misalignment between what is taught and what is assessed  

---

# 5. Misconception-Aware Design

Design should anticipate and address likely learner misunderstandings.

### Requirements

- Identify plausible misconceptions from content.
- Use misconceptions to:
  - inform distractors (in MCQs)  
  - shape feedback  
  - guide activity design  

- Misconceptions must be:
  - plausible  
  - grounded in the source material  

---

# 6. Effective Feedback

Feedback should support **learning improvement**, not just judgement.

### Requirements

- Feedback must:
  - explain why an answer is correct or incorrect  
  - connect back to underlying concepts  

- Feedback should be:
  - concise  
  - actionable  
  - directly linked to learner errors  

- Prefer scalable feedback models suitable for large cohorts.

---

# 7. Cognitive Load Management

Content and tasks should be structured to **reduce unnecessary cognitive load**.

### Requirements

- Break content into manageable chunks.
- Avoid unnecessary complexity in:
  - language  
  - task instructions  
  - question design  

- Maintain clarity and focus on key ideas.

---

# 8. Content Fidelity

All generated outputs must remain **faithful to the source content**.

### Requirements

- Do not introduce:
  - new concepts  
  - external knowledge  
  - unsupported claims  

- All:
  - concepts  
  - questions  
  - feedback  

must be grounded in the input content or derived artefacts.

---

# 9. Scalability

Designs should be suitable for **large-scale delivery**.

### Requirements

- Avoid reliance on:
  - intensive tutor intervention  
  - bespoke marking  

- Prefer:
  - automated assessment  
  - reusable activities  
  - structured outputs  

---

# 10. Accessibility and Clarity

Outputs must be **clear, inclusive, and usable**.

### Requirements

- Use clear, direct language.
- Avoid unnecessary jargon unless defined.
- Ensure instructions and questions are unambiguous.

---

# 11. Instructional Executability

Learning designs must be directly runnable in real teaching contexts without requiring additional redesign.

### Requirements

- Activities must include:
  - concrete tasks
  - required inputs/materials
  - observable learner outputs
  - enough specificity to run directly

- Sequences must include:
  - timing
  - transitions
  - facilitator actions where needed

- Avoid:
  - abstract or descriptive-only activities
  - outputs that require facilitator invention before use

---

# How to Apply These Principles in Workflows

When generating or evaluating workflows:

- Use these principles to:
  - decide which steps to include  
  - shape prompt instructions  
  - validate outputs  

- If a trade-off is required:
  - prioritise alignment, validity, and clarity  

---

# Summary

These principles define what “good learning design” means within PRISM.

They should be treated as:

> **non-optional constraints that guide workflow design and output quality**
