# Sprint 52 Charter — Production Quality Resource

**Status:** Open  
**Predecessor:** Sprint 51 — Learner Experience Quality (closed)  
**Operating manual:** [SPRINT-52-CONTEXT.md](./SPRINT-52-CONTEXT.md)

---

## Mission

Take the current PRISM learner page and make it an **excellent learner-facing resource**.

Sprint 50 established **how** pedagogy manifests. Sprint 51 added **expert judgement, evaluative coaching, preservation, and salience**. Sprint 52 asks whether the **whole resource** meets production quality — consistently, across activities, with excellent teaching voice.

---

## What Sprint 52 Is

| Yes | Meaning |
| --- | ------- |
| Production-quality learner experience | Voice, depth, consistency, coherence |
| Evidence-led quality slices | Finding → narrow fix → fresh re-walk |
| Cross-activity audit | Every activity held to the same bar |
| Targeted generation/content improvements | Within existing fields and patterns |
| Renderer polish | Salience and hierarchy refinements when evidence supports |

## Success Criterion

A learner opening a **freshly generated** resource can:

- Understand **why** each activity exists — without guessing  
- Know **how** to think before doing — consistently  
- Learn from **models** with appropriate depth — not uneven quality  
- **Check** work with clear criteria and coaching — not arbitrary tick-lists  
- **Reflect** and **transfer** authentically — where the design calls for it  
- Experience **one coherent teaching voice** across the whole resource  

Success is measured by **rendered learner walkthroughs on fresh runs** — not field inventories or stale artefacts.

---

## Investigation Approach

1. **Fresh run first** — current codebase, exported HTML  
2. **Activity-by-activity audit** — all activities, not just A1  
3. **Rubric before slices** — document criteria before code  
4. **One finding → one slice** — minimal diffs  
5. **Regression guard** — Sprint 50 + Sprint 51 suites remain green
