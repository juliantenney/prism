# Sprint 64 — Starting Assumptions

## Assumptions Supported By Sprint 63

* Activities differ cognitively; homogeneous presentation can hide that.  
* Tier 2 manifestation from render-time signals works for visibility gains.  
* Priority-1 archetype plans contain learner-relevant intermediate reasoning structure.  
* Some of that structure cannot be safely recovered downstream.  
* Flattening occurs at GAM → assembled `materials[]` on production-like paths.  
* Architecture investigation is justified; schema redesign is not (on current evidence).  
* Verbatim plan surfacing has manageable semantic risk; paraphrased invention does not.

---

## Assumptions Not Yet Supported

* Any specific preservation attachment point is best.  
* Any specific manifestation contract is best.  
* Preservation improves measured learning outcomes.  
* Production implementation is proportionate.  
* All four Priority-1 archetypes behave identically (`evaluation_judgement` not re-validated in Experiment 3).  
* A production change should be authorised after this sprint (investigation may still recommend “wait”).

## Questions Sprint 64 Must Answer

1. What candidate preservation mechanisms exist on the **current** pipeline without schema redesign?  
2. Where could preserved structure attach (preservation location)?  
3. What candidate manifestation contracts could consume preserved structure safely?  
4. Where would manifestation decisions occur (before vs during render)?  
5. What are the trade-offs (technical, semantic, coupling, maintenance)?  
6. What is the **smallest viable** combination worth recommending next?  
7. Is the recommendation “investigate further,” “authorise a bounded prototype,” or “do not proceed”?
