# Sprint 62 — Renderer Coherence Checklist

Use this checklist for each targeted renderer pass page. For Sprint 62, apply to all four baseline pages in [acceptance-evidence-pages.md](acceptance-evidence-pages.md) unless a change is narrowly scoped and justified.

## Learner coherence checks

- [ ] Learner does not see generic "What to do" wrappers that duplicate surrounding text.
- [ ] Learner does not see generic "Output" wrappers that add no pedagogical value.
- [ ] Learner-facing metadata labels are hidden or transformed into meaningful prose.
- [ ] Instructions and materials read in coherent narrative progression.
- [ ] Repeated instructional labels/preambles are removed where redundant.

## Safety checks

- [ ] All educational content remains present.
- [ ] Deterministic ordering remains intact.
- [ ] Pedagogical sequence remains intact.
- [ ] No DLA/GAM/archetype/assessment/persistence/assembly architecture change.

## Evidence checks

- [ ] Before/after renderer snapshots retained.
- [ ] A concise rationale is recorded for each changed pattern.
