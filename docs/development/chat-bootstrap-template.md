# PRISM Chat Bootstrap Template

## Current State

Short summary of current platform state.

## Current Priorities

What consolidation/review work is active.

## Current Constraints

Examples:

- consolidation phase
- avoid uncontrolled feature expansion
- preserve runtime compatibility
- preserve domain-pack filename stability

## Relevant Files

Only include files directly relevant to the next bounded task.

## Recent Decisions

Most relevant architectural decisions.

## Immediate Task

Single coherent next objective.

## Suggested Focus

Clarify what NOT to drift into.

## Recommended Sprint Folder

The sprint folder should include only files needed to bootstrap the next bounded chat.

Required files:

- `SPRINT-CONTEXT.md`
- `GPT-BOOTSTRAP-PROMPT.md`
- `CURRENT-STATE.md`
- `HANDOVER.md`
- `context-files/` (bounded implementation/docs files only)

Optional files:

- one or two directly relevant architecture/spec docs
- one recent decision record if needed for constraints

Excluded areas:

- unrelated modules or historical files
- broad exploratory docs not needed for the bounded task
- full-project exports, backups, or archive sweeps

## Suggested Opening Prompt

Use this concise starter format for the next chat:

"Context: [current state + constraints]. Task: [single bounded objective]. Scope: [exact files/areas]. Non-goals: [what to avoid]. Success criteria: [verification outcomes]."
